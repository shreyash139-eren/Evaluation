Client & Server communication


In the field of software development by Client side we mean the Front-End of the web application, whereas Back-End is the server side of the Web application. Client and Server communicate by verifying the data entered by the user in the Client side, which is Front-End and then the data provided by user is send to the server for verification.


After recieving the data from the user server verifies the credentials if there is any and if the credentials are correct then it grants access to the user using API's. Some Websites allows you to search for something without any credentials like Google, It transfers the data related to the keywords entered by the user from a vast database to the screen.

The Journey of a Web Request


A web requests journey starts with the user providing some credential or data to process in browser and after that the request is sent to the server side of the Web page by browser to be processed according to the request. If the data sent to the server is credentials for log in, then it will verify the username and password from the database and if both matches then only the server will redirect the user to its dashboard or homepage through the browser, and if the request recieved by the server is for some content, then it will show the contents related to the keywords provided by the user from the database using API's. Google is the leading example of search engines.

Client vs Server
Function Client Server
Purpose Requests and receives services from a server. Provides services and resources to clients.
Initiation Initiates requests to the server (e.g., browsing a website). Responds to client requests and performs necessary actions.
Resource Access Accesses resources (data, services, etc.) provided by the server. Manages and provides resources to clients.
Control Typically has less control over server-side operations. Controls the server's resources and services.

Static vs Dynamic Websites
Feature Static Dynamic
Content Generation Pre-built, static HTML files Generated on-the-fly based on user requests
Interactivity Limited interactivity, primarily fixed content Higher interactivity, personalized experiences
Technologies HTML, CSS, JavaScript HTML, CSS, JavaScript, server-side languages (PHP, Python, etc.)


Quick facts and tips:

  • The Internet is not owned or controlled by any single entity.
  • Each device on the network has a unique IP address, which helps routers know where to send the packets.
  • Data is broken down into small packets, like pieces of mail, and sent over the network.
